# Документы (КП, письма, сметы)

Канон оформления фирменных Word/PDF-документов. Генерация КП автоматизирована — скилл **dicom-kp** (`scripts/build_kp.js`, контент JSON → брендированный .docx + PDF).

## Типографика документов

- Шрифт — **Arial** (универсально открывается у клиента; Roboto в docx не используем во избежание подстановок).

## Цвета (из брендбука)

Акцент — синий `#0D75B2` (НЕ голубой Битрикс24 `#0DACE5`). Заголовки H1 — тёмный `#0F1B33`.

## Фирменное оформление

- **Шапка** — логотип DICOM + тэглайн «DIgitise your COMpany · Битрикс24 · ИИ-решения · веб-разработка». Верхний колонтитул пустой.
- **Заголовки** — `#0F1B33` с акцентной линией снизу (`#0D75B2`).
- **Таблицы** — шапка на акценте `#0D75B2` белым текстом; зебра строк `#F6F7F9`; итог — подложка `#DDEAF5`; рамки `#E5E7EB`.
- **«Состав работ по этапам»** — вертикальный process-таймлайн: кружки `#DDEAF5` с акцентной цифрой `#0D75B2`, связующая линия `#E5E7EB`, пилюля часов справа.
- **Подвал** — тёмная плашка `#0F1B33`: белый логотип, тэглайн, копирайт, колонки «Контакты»/«Руководитель» (без мессенджеров и без синей полоски сверху).

## Тон

По-деловому, конкретно, без воды: «понятный результат, а не презентация», глаголы дела («разработаем», «настроим», «сдадим»), цифры вместо эпитетов. Подробнее — [`../../voice-tone/voice.md`](../../voice-tone/voice.md).

> Структура и поля КП, генератор и грабли — в скилле dicom-kp (`references/structure.md`, `references/generate.md`).
